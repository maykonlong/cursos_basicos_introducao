Curso Completo de Postman

1. O que é uma API?

Interface de Programação de Aplicativos

Uma API é um conjunto de regras e ferramentas que permite que diferentes aplicativos de software se comuniquem entre si. As APIs atuam como intermediárias, possibilitando a troca de dados entre sistemas.

APIs de Clima

Para exibir previsões do tempo em aplicativos

APIs de Pagamento

Como PayPal para processar transações

APIs Sociais

Para integrar com redes sociais


Cliente ↔ API ↔ Servidor

2. O que é o Postman?


Interface do Postman

Ferramenta Poderosa para APIs

O Postman é uma ferramenta completa para testar, documentar e automatizar interações com APIs. Ele simplifica o processo de envio de requisições HTTP e análise de respostas.

  • Interface gráfica intuitiva
  • Suporte a vários métodos HTTP
  • Automação de testes
  • Colaboração em equipe
  • Versão gratuita disponível

3. Visão Geral sobre HTTP

O HTTP é o protocolo usado para comunicação na web. Ele define como as requisições são enviadas e como as respostas são retornadas.

Métodos HTTP

GET
Recupera dados
POST
Cria recursos
PUT
Atualiza recursos
DELETE
Remove recursos

Códigos de Status

2xx - Sucesso

200 (OK), 201 (Criado)

4xx - Erro do Cliente

400 (Requisição inválida), 404 (Não encontrado)

5xx - Erro do Servidor

500 (Erro interno do servidor)

4. Interface do Postman

URL e Métodos

Campo para inserir o endereço da API e dropdown para selecionar o método HTTP (GET, POST, etc.)

Parâmetros

Permite adicionar query parameters e configurar a estrutura da requisição

Autorização

Configura diferentes tipos de autenticação: Basic Auth, Bearer Token, OAuth, etc.

Headers & Body

Define metadados da requisição e envia dados no formato desejado (JSON, form-data, etc.)

Scripts

Pré-Request scripts executados antes da requisição e Tests executados após a resposta

Configurações

Ajustes gerais como certificados SSL, timeouts e outras preferências

5. Como Criar uma Requisição

  • Clique em New > Request
  • Dê um nome à requisição (ex.: "Obter Usuários")
  • Selecione o método HTTP (ex.: GET)
  • Insira a URL da API (ex.: https://api.exemplo.com/users)
  • Configure parâmetros, headers ou body, se necessário
  • Clique em Send para enviar a requisição
  • Analise a resposta na seção inferior (status, corpo, headers)

    • Processo de Criação de Requisição

    6. Como Importar uma Coleção

    Coleções Compartilháveis

    Uma coleção é um conjunto de requisições salvas que podem ser compartilhadas ou reutilizadas entre equipes e projetos.

  • Clique em Import no canto superior esquerdo
  • Escolha a fonte: Arquivo JSON, URL ou definição cURL
  • Faça upload do arquivo ou cole o conteúdo
  • Confirme a importação e veja a coleção na barra lateral

    • Importação de Coleções

    7. Como Instalar um Certificado


    Configuração de Certificados SSL

    Autenticação por Certificado

    Para APIs que requerem autenticação por certificado SSL, siga estes passos:

  • Vá para Settings (ícone de engrenagem) > Certificates
  • Clique em Add Certificate
  • Insira o domínio da API (ex.: api.exemplo.com)
  • Faça upload dos arquivos .crt (certificado) e .key (chave privada)
  • Confirme e ative o certificado

    • 8. Como Utilizar Variáveis

    As variáveis no Postman permitem reutilizar valores, como URLs ou tokens, sem repeti-los manualmente.

    Variáveis Globais

    Acessíveis em todas as coleções e ambientes do workspace

    Variáveis de Coleção

    Específicas para uma coleção particular

    Variáveis de Ambiente

    Específicas para um ambiente (produção, teste, desenvolvimento)

    // Exemplo de uso de variáveis
{{base_url}}/users/{{user_id}}

// Definir variável em script
pm.environment.set("user_id", "123");

// Obter variável em script
const userId = pm.environment.get("user_id");

    9. Post-Response e suas Funções

    Scripts de Validação

    Os scripts de Post-Response (aba Tests) são executados após a resposta da API. Eles são usados para validar respostas ou extrair dados para uso posterior.

    • Validação de status codes
    • Verificação de conteúdo
    • Extração de dados
    • Definição de variáveis

    Testes Automatizados 
    // Validar status da resposta
pm.test("Status é 200", function () {
    pm.response.to.have.status(200);
});

// Extrair dados da resposta
const response = pm.response.json();
pm.environment.set("user_id", response.id);

// Validar conteúdo JSON
pm.test("Resposta contém nome", function () {
    pm.expect(response.name).to.not.be.undefined;
});

    10. Pré-Request e suas Funções


    Scripts Pré-Requisição

    Preparação Automática

    Os scripts de Pré-Request são executados antes da requisição para configurar headers, tokens, timestamps ou outros dados necessários.

    // Definir timestamp atual
pm.environment.set("timestamp", Date.now());

// Configurar token de autenticação
pm.environment.set("auth_token", "Bearer " + pm.environment.get("api_key"));

// Gerar dados dinâmicos
const randomId = Math.floor(Math.random() * 1000);
pm.environment.set("random_id", randomId);

    11. Geração de Código

    Export para Outras Linguagens

    O Postman permite gerar trechos de código para várias linguagens a partir de uma requisição configurada.

  • Crie e configure sua requisição
  • Clique no ícone Code (</>) à direita da URL
  • Selecione a linguagem desejada (Python, JavaScript, cURL, etc.)
  • Copie o código gerado para seu projeto

    • Suporte Multi-linguagem

    Python

    Requests, urllib

    JavaScript

    Fetch, Axios, jQuery

    cURL

    Comando para terminal

    Java

    OkHttp, Unirest

    12. Exercícios Práticos

    Pratique seus conhecimentos com estes exercícios hands-on

    1

    Criar uma Requisição GET

    Objetivo: Criar uma requisição para a API pública JSONPlaceholder

    Passos a seguir:

    1. Abra o Postman e clique em New > Request
    2. Nomeie a requisição como "Listar Usuários"
    3. Selecione o método GET
    4. Insira a URL: https://jsonplaceholder.typicode.com/users
    5. Clique em Send e verifique se o status retornado é 200
    6. Na aba Tests, adicione o script de validação
    7. Salve a requisição em uma coleção chamada "API Testes"
    // Script para aba Tests
pm.test("Verificar status 200", function () {
    pm.response.to.have.status(200);
});

pm.test("Resposta é um array", function () {
    pm.expect(pm.response.json()).to.be.an('array');
});
    2

    Criar uma Requisição POST

    Objetivo: Criar um novo usuário na API JSONPlaceholder

    Passos a seguir:

    1. Crie uma nova requisição chamada "Criar Usuário"
    2. Selecione o método POST
    3. Insira a URL: https://jsonplaceholder.typicode.com/users
    4. Na aba Body, selecione raw e escolha JSON
    5. Insira o JSON do usuário
    6. Envie a requisição e verifique o status 201
    7. Adicione script para extrair o ID da resposta
    Body JSON:
    {
    "name": "João Silva",
    "email": "joao.silva@example.com",
    "username": "joaosilva"
}
    Script Tests:
    const response = pm.response.json();
pm.environment.set("novo_usuario_id", response.id);

pm.test("Status é 201", function () {
    pm.response.to.have.status(201);
});
    3

    Converter cURL para Postman

    Objetivo: Importar um comando cURL para o Postman

    Comando cURL:

    curl -X GET "https://jsonplaceholder.typicode.com/posts?userId=1" \
-H "Accept: application/json"

    Passos:

    1. Clique em Import no Postman
    2. Selecione a aba Raw Text
    3. Cole o comando cURL acima
    4. Clique em Continue e revise a requisição
    5. Envie a requisição e confirme status 200
    6. Salve na coleção "API Testes"
